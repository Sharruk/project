🔥 ATM SYSTEM — ENHANCED (HIGH-SCORING VERSION)
🟢 1. Problem Statement (slightly improved)

An Automated Teller Machine (ATM) system is required to provide secure and automated banking services such as cash withdrawal, deposit, balance enquiry, and fund transfer. The system must ensure fast processing, data accuracy, and continuous availability while minimizing manual intervention.

🟢 2. Stakeholders (add clarity)
Customer – Performs transactions like withdrawal, deposit, enquiry.
Bank/Admin – Maintains accounts, monitors transactions, ensures security.
Operator – Maintains ATM machine (cash refill, system control).
System – Processes transactions and updates database automatically.
🟢 3. Functional Requirements (keep clean but strong)
System shall allow user to withdraw cash in multiples of 100.
System shall allow user to deposit cash.
System shall allow user to check account balance.
System shall allow user to transfer funds between accounts.
System shall print transaction receipts.
System shall validate PIN before transactions.
System shall record all transactions in database.
System shall allow user to perform multiple transactions per session.
🟢 4. Non-Functional Requirements (refined)
Security – Validate PIN and block card after 3 failed attempts.
Performance – Each transaction must complete within 60 seconds.
Reliability – Ensure accurate and consistent transaction records.
Usability – Provide simple keypad and display interface.
Availability – System must operate 24/7.
Session Control – Timeout after inactivity (2 minutes).
Concurrency – Serve one customer at a time.
🟢 5. Constraints (tight + clean)
Must support ATM hardware (card reader, dispenser, printer).
Must allow operator control (start/stop system).
Must enforce transaction limits and banking policies.
🟢 6. DFD (don’t change structure — just cleaner wording)
🔹 Level-0

Customer → (0.0 ATM System) → Cash / Receipt
Bank → (0.0 ATM System)

🔹 Level-1

1.0 Authenticate User
2.0 Process Transaction
3.0 Update Account

Data Stores:
D1 Account DB
D2 Transaction DB

🔹 Level-2 (1.0)

1.1 Insert Card
1.2 Enter PIN
1.3 Validate PIN

🟢 7. Architecture (slightly better wording)

Layered Architecture

Presentation Layer → ATM UI (screen + keypad)
Application Layer → Transaction logic & validation
Data Layer → Account & transaction database

Modules:

Authentication Module
Transaction Module
Database Module
Interface Module
