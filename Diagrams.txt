🔹 1. Presentation Layer

🔹 2. Business Layer
This layer contains core logic of the system.

Booking Module handles appointment booking.

This layer interacts with the database.

🔹 4. Database Layer
This layer stores all system data including customer details, booking records, services, and staff information.

USE CASE DIAGRAM


🔥 WHAT USE CASE DIAGRAM REALLY MEANS (NO CONFUSION)

Forget theory. Think like this:

👉 Use Case = What user does with system
👉 Actor = Who uses system

That’s it.

🟢 1. ACTOR (Stick Man)

Meaning:

Actor = external user interacting with system

In your question:

Customer
Staff
Manager/Admin
🟢 2. USE CASE (Oval)

Meaning:

Use Case = action performed in system

From your question:

Login
Search Service
Book Service
Confirm Booking
Update Service Status
Generate Report
🟢 3. ASSOCIATION (Line)

Meaning:

Connects actor to use case

Example:

Customer ——> Book Service

👉 Just a straight line. Nothing fancy.

🟢 4. <<include>> (IMPORTANT)

Meaning:

One process ALWAYS uses another process

Example:

Book Service  <<include>>  Check Availability

👉 Booking ALWAYS needs availability check.

🟢 5. <<extend>> (IMPORTANT)

Meaning:

Optional behavior (extra condition)

Example:

Book Service  <<extend>>  Suggest Alternative Slot

👉 Only happens if slot is not available.

🟢 6. GENERALIZATION (OPTIONAL)

Meaning:

One actor is a type of another

Example:

User → Customer / Admin

👉 You can skip this if no time.
